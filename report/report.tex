\begin{abstract}
This report presents a comprehensive data mining and analysis study conducted on large-scale Amazon datasets. The project is divided into two complementary parts. Part 1 mainly focuses on Exploratory Data Analysis (EDA) and Sentiment Analysis to explore user reviews, ratings and the overall structure of the datasets. Part 2 involves text pre-processing, clustering of products to group them into subsets, a recommendation system and frequent pattern mining to uncover association rules and purchasing patterns. The work is focused on searching (mining) for useful and interesting information on large-scale datasets, the characteristics of which we do not know a priori. The Amazon product categories that were used are the following five: Appliances, All\_Beauty, Musical\_Instruments, Software and Video\_Games.
\end{abstract}

\subsection{Data Pre-Processing}
\begin{itemize}
    \item All missing price values are replaced with the mean price value.
    \item All bad values or NULL values are replaced with the mean price value.
    \item All HTML elements are removed from the text column of the reviews(e.g., \texttt{\&amp;}).
\end{itemize}
After experimentation, it was found that no other pre-processing was necessary for this part.
An important design choice during data cleaning was the robust handling of missing or corrupted price values:

\begin{lstlisting}[language=Python]
def replace_invalid_prices(df, mean):
    df['price'] = pd.to_numeric(df['price'], errors='coerce') 
    df.loc[(df['price'].isna()) | (df['price'] <= 0.0), 'price'] = mean
    return df
\end{lstlisting}

By utilizing the \texttt{errors='coerce'} parameter, any corrupted string values are safely converted to \texttt{NaN}. Furthermore, the condition \texttt{df['price'] <= 0.0} ensures that free or negatively priced items---which are typically scraping artifacts or errors in this context---are also treated as missing. Replacing these invalid entries with the category's mean (mean imputation) preserves the dataset's volume for subsequent text analysis without skewing the expected price distribution.

Next are the best sellers of each category. Such products have a high review count and a high average rating score.
To accurately identify the ``best-selling'' products, a naive sorting by average rating or review count is insufficient. Sorting solely by rating often surfaces obscure products with a single 5-star review, while sorting solely by review count highlights popular but potentially mediocre items. To solve this, a multi-stage ranking heuristic was implemented:

\begin{lstlisting}[language=Python]
# Keep only products with high rating
merged_df = merged_df[merged_df['avg_rating'] >= 4]

# Sort by verified review count descending and take top_n * 2
top_products = merged_df.sort_values(by='verified_review_count', ascending=False).head(top_n * 2)

# From those highly reviewed products, sort by avg_rating 
top_products = top_products.sort_values(by='avg_rating', ascending=False).head(top_n)
\end{lstlisting}

This approach guarantees statistical significance by first isolating a larger pool (\texttt{top\_n * 2}) of the most heavily reviewed, well-rated products, and then extracting the absolute best based on their final average rating.

\subsection{Sentiment Analysis Methodology}
In this section, an attempt is made to capture the sentiment of all reviews, in order to ultimately compare the textual content of the product reviews to their respective ratings. The pre-trained model used for sentiment analysis is \href{https://vadersentiment.readthedocs.io/en/latest/#}{VADER} (Valence Aware Dictionary and sEntiment Reasoner).
Initially, a sentiment extraction function is applied to all reviews ('text' column) populating the new 'vader\_sentiment' column with floating-point values ranging between -1 and 1. If the user's rating for a product is at least 4/5, then 0.2 is added to the sentiment score whereas if the rating is no more than 2/5, 0.2 is subtracted from the sentiment score. These adjusted sentiment scores are stored in a new column: 'adjusted\_sentiment'. This adjustment is crucial because it amplifies the positive and negative reviews making the classification easier. Finally, reviews with adjusted sentiment scores ranging from -1 to -0.33333 are labeled as negative. Symmetrically, scores between 0.33333 and 1 are labeled as positive, while the rest are neutral reviews. A sample output of the above process follows.\newline

An important optimization was implemented in the sentiment extraction function:
\begin{lstlisting}[language=Python]
analyzer = SentimentIntensityAnalyzer()
def compute_vader_sentiment(text):
    if pd.isna(text):
        return 0.0
    return analyzer.polarity_scores(str(text)[:1000])['compound']
\end{lstlisting}
By applying the string slice \texttt{[:1000]}, only the first 1000 characters of each review are processed to extract the sentiment. This choice conserved significant computational resources and time, as the datasets contained some extremely long reviews. Furthermore, since VADER's algorithm relies on lexicon and rule-based heuristics, evaluating the first 150 to 200 words (roughly 1000 characters) is generally more than sufficient to accurately capture the overall polarity of a text.

\section{Part 2: Product Analysis, Clustering and Market Basket Mining}

\subsection{Introduction to Part 2}
In this second part of the project, the focus shifts towards unsupervised machine learning techniques and data mining algorithms to discover hidden patterns within the product metadata and user transactions. Specifically, the text features of the products' titles and descriptions are processed and vectorized to cluster similar products together. Following this, a recommendation system is designed, both user-based and content-based. Finally, frequent pattern mining is applied to user purchasing histories to uncover association rules and seasonal market trends, with a special focus on the month of December as an example.

\subsection{Dataset Preparation and Text Pre-Processing}
Before applying any machine learning models, the textual metadata of the products (such as titles and descriptions) must be transformed into a clean, standardized format. The pre-processing pipeline consists of the following steps:
\begin{itemize}
    \item \textbf{Text Normalization:} All characters are converted to lowercase, and punctuation marks are completely removed to ensure uniformity.
    \item \textbf{Expansion of Contractions:} Common English contractions (e.g., "don't" to "do not") are expanded using the \texttt{contractions} library.
    \item \textbf{Spell Correction:} The \texttt{pyspellchecker} library is utilized to correct misspelled words, reducing the noise in the dataset.
    \item \textbf{Stopword Removal:} Frequently used words that carry minimal semantic weight (e.g., "the", "is", "in") are removed.
\end{itemize}

\subsection{Feature Extraction}
Following the text pre-processing, the clean text is transformed into numerical representations suitable for machine learning algorithms. This is achieved using the Term Frequency-Inverse Document Frequency (TF-IDF) vectorizer. TF-IDF evaluates how relevant a word is to a document within a collection of documents, assigning higher weights to terms that are frequent in a specific product's description but rare across the entire category. This step successfully converts the textual data into high-dimensional numerical vectors.

\subsection{Product Clustering}

\subsubsection{Clustering Methodology}
To group similar products together, the K-Means clustering algorithm is applied to the TF-IDF feature space. An important observation during this phase is the relatively low silhouette scores and the visual overlap between clusters. 

This behavior is expected and stems directly from the nature of the data. Because clustering relies on Natural Language Processing (NLP) over product descriptions, texts frequently share overlapping semantics and vocabulary. Furthermore, the TF-IDF vectorization creates an extremely high-dimensional feature space. In such spaces, distance metrics like the Euclidean distance lose their discriminative power---a statistical phenomenon widely known as the ``Curse of Dimensionality.'' Consequently, the clusters blend into one another naturally, rather than forming strictly separated boundaries.

\subsubsection{Cluster Selection}
To determine the optimal number of clusters ($k$), the Elbow Method is utilized alongside the \texttt{kneed} Python library, which mathematically identifies the point of maximum curvature. An important design choice was made regarding the search space for $k$:

\begin{lstlisting}[language=Python]
# The search range for optimal k is intentionally restricted
k_values = range(5, 15)
\end{lstlisting}

This constraint was implemented because initial experiments revealed that without a lower bound, the algorithm occasionally identified $k=4$ as optimal for certain categories, but produced highly imbalanced or almost empty clusters. By enforcing a minimum of 5 clusters, the algorithm distributes the high-dimensional data more evenly, ensuring that the resulting subsets are dense and meaningful for the subsequent recommendation system.

\subsubsection{Cluster Interpretation}
Once the clusters are formed, they are qualitatively analyzed. By extracting the top TF-IDF terms for each cluster, distinct themes emerge, allowing us to assign meaningful labels to them.

\subsection{Cluster-Based Recommendation Strategy}
A recommendation system was developed utilizing the previously formed clusters. When a user interacts with a specific product, the system first identifies the cluster to which this product belongs. Then, it calculates the Cosine Similarity between the target product's TF-IDF vector and the vectors of all other products within that same cluster. By restricting the similarity search exclusively within the cluster boundaries, the recommendation system operates significantly faster and suggests highly relevant products.

\subsection{Association Rule Mining}
To uncover hidden purchasing patterns, Market Basket Analysis is performed using the FP-Growth algorithm from the \texttt{mlxtend} library. A critical optimization was implemented here to prevent memory overflow (RAM crashes) when processing massive datasets:

\begin{lstlisting}[language=Python]
# Dynamic parameter calculation based on dataset size
min_item_frequency = max(15, int(len(current_df) * 0.0002))
min_support = 0.001 * (1 + (len(current_df) // 3_000_000))

# Removing duplicate rules (A->B and B->A)
rules_all['frozen_pair'] = rules_all.apply(
    lambda row: frozenset(row['antecedents'].union(row['consequents'])), axis=1
)
rules_unique = rules_all.drop_duplicates(subset=['frozen_pair'])
\end{lstlisting}

Rather than using hardcoded limits, the \texttt{min\_item\_frequency} and \texttt{min\_support} parameters scale dynamically based on the dataset's size (e.g., keeping only the top 0.02\% of products). This prevents the FP-Tree from exploding with noisy, low-frequency itemsets in datasets exceeding 4 million rows (such as Software and Video Games). Additionally, since the Lift metric is mathematically symmetric, the algorithm inherently produces duplicate pairs (A $\rightarrow$ B and B $\rightarrow$ A). Using \texttt{frozenset} effectively filters out these duplicates, presenting only unique and actionable rules.
